This page shows the results of the Crate.IO benchmarks based on the master branch (nightly builds) as of that point in time. The benchmarks are executed as JUnit benchmarks on Microsoft Azure cloud environment Standard_DS12 instances (4 cores 2.4 GHz Intel Xeon® E5-2673 v3 Haswell machine with 28 GB RAM and SSD).

The benchmark setup is as follows: 2 Crate Nodes, running with default settings (except following) on a single host:

  • index.store.type: memory, ensures that the index is stored in main memory
  • CRATE_HEAP_SIZE is set to 10 GB

BulkInsertBenchmark

Tests bulk operations with insert and delete statements. Bulk operations are executed as single calls on the server. It can be compared to prepared statement.



BulkDeleteBenchmark



InsertBenchmark

Tests the performance of single and bulk inserts of Crate compared to Elasticsearch API.



CountBenchmark

Tests the performance of COUNT aggregation function of Crate compared to Elasticsearch API.



GroupByBenchmark

Compares different tests with GROUP BY aggregation function on Crate.



GroupByArbitraryBenchmark

Compares test results using standard GROUP BY to ARBITRARY aggregation function.



CrossJoinBenchmark

Shows results of different test cases which use the CROSS JOIN functionality.



AnyBenchmark

Shows results of test cases that use the ANY operator.



LikeBenchmark

Tests the performance of LIKE operator of Crate compared to Elasticsearch API.



InStringBenchmark

Shows test results of queries that use the IN operator on columns of type STRING.



InNumericBenchmark

Shows test results of queries that use the IN operator on columns of type NUMBER.



ESScrollingBenchmark

Shows test results of the Elasticsearch Scroll API.



BaseCreateBenchmark

Tests the query performance of creating a single table in Crate.



CreateWith10TablesBenchmark

Tests the query performance of creating 10 tables in Crate.



CreateWith200TablesBenchmark

Tests the query performance of creating 200 tables in Crate.



InformationSchemaBenchmark

Tests the performance for fetching results of the INFORMATION_SCHEMA.



AddingTableColumnsBenchmark

Show test results of ALTER TABLE statement for different amount of columns.
