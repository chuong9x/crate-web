title: Bulk Inserts with unnest link: https://crate.io/a/bulk-inserts-with-unnest author: Claus Matzinger description: unnest - A handy table function for bulk inserts. created: 2016-06-13 post_name: bulk-inserts-with-unnest status: publish post_type: post tags: example, feature category: news, developernews Table functions are functions that produce rows in the same way as a table, and with Crate's latest testing release (0.55.0) we have added support for [unnest](/docs/reference/en/latest/sql/table_functions.html). `Unnest` uses arrays to generate a transient table with each array representing a column: ```sql cr> select * from unnest([1, 2, 3], ['Arthur', 'Trillian', 'Marvin']); +------+----------+ | col1 | col2 | +------+----------+ | 1 | Arthur | | 2 | Trillian | | 3 | Marvin | +------+----------+ SELECT 3 rows in set (... sec) ``` ## Bulk Inserts To insert large amounts of data efficiently, you are recommended to use [bulk inserts](/docs/reference/sql/rest.html#bulk-operations). With a client that does not support these or if you have no direct access to the rest endpoint you can be out of luck! The same is also unfortunately true for our [ODBC driver](/enterprise/odbc) since the ODBC specification does not have a way to perform bulk requests. With `unnest` there is a way around this by using [insert-by-query](/docs/reference/sql/dml.html#inserting-data-by-query) you can select from an 'unnested' table which results in a bulk insert internally: ```sql cr> insert into authors (id, name) (select * from unnest([1, 2, 3], ['Arthur', 'Trillian', 'Marvin'])); INSERT OK, 3 rows affected (... sec) ``` As the response suggested, the three rows have now been inserted: ```sql cr> select * from authors order by id; +----+----------+ | id | name | +----+----------+ | 1 | Arthur | | 2 | Trillian | | 3 | Marvin | +----+----------+ SELECT 3 rows in set (... sec) ``` The resulting performance is also comparable to bulk requests ![Bulk inserts vs Unnest]({% media '/media/1606/unnest-vs-bulk.png' %}) *Comparison of a single bulk request and insert via 'unnest' of 1000 rows, repeated 10 times on a single machine.* We have extended the [crate-version-comparison Github repository](https://github.com/crate/crate-version-comparison) to include a way to insert with unnest and compare single-threaded performance. ## Try it Out!
